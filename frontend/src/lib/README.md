# lib/

Cross-cutting infrastructure with no feature logic of its own: the HTTP transport
layer and shared UI helpers. Two files.

## `api.ts` — the axios transport layer

The configured axios instance plus everything that wraps every request:

- **`api`** — the axios client (base URL from `VITE_API_BASE_URL`).
- **`tokenStore`** — the single chokepoint for the JWT in `localStorage`, so no
  other module hardcodes the storage key. The request interceptor reads it; auth
  writes it; logout and the 401 handler clear it.
- **Interceptors** — the request interceptor attaches the bearer token; the
  response interceptor centralizes 401 handling and normalizes every failure.
- **`ApiError` / `ProblemDetails`** — `ProblemDetails` mirrors the backend's
  RFC 7807 error envelope; `ApiError` flattens it into a `.message` + `.status`
  that every caller can rely on, instead of a raw `AxiosError`.
- **`projectsApi` / `usersApi`** — the raw async request methods, one per
  endpoint, each parsing its response through the matching Zod schema.

### Two cross-cutting guarantees

Every caller downstream of this file can assume:

1. **Responses are validated.** Each method `.parse()`s through its schema, so
   callers receive fully-typed data — date fields are real `Date` objects.
2. **Failures are normalized.** Rejections are always `ApiError`, never a raw
   axios error, so error handling is uniform (`err.status === 404`, `err.message`).

### The two `api`-named layers — don't conflate them

There are two things called "api" in this codebase, and they're different layers:

- **`src/lib/api.ts`** (this file) — axios instance + raw request methods.
- **`src/api/*.ts`** — the TanStack Query hooks that _wrap_ these methods with
  caching and invalidation.

Pages call the hooks in `src/api/`, not these raw methods directly.

### What's deliberately absent

Auth endpoints (`/api/auth/login`, `/api/auth/register`) are **not** here — they
live with the auth store (`src/auth/AuthContext.tsx`), because they mint the very
token the rest of these calls depend on.

### One cross-cutting behavior to know

On a 401, the response interceptor clears the token and hard-redirects to
`/login` via `window.location.href` — not router navigation. The full reload is
deliberate: it guarantees a clean auth reset (in-memory state and the TanStack
cache both wiped) that a soft navigation wouldn't. A path guard prevents a
redirect loop when the 401 originated on the login page itself.

## `utils.ts` — shared UI helpers

Just `cn`, the standard shadcn/ui class merger (`clsx` for conditional classes +
`tailwind-merge` to resolve conflicting Tailwind utilities, last-wins).
Generated by shadcn, left as-is.
